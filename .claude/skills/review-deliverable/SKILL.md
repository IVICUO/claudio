# Review deliverable

Apply the IVICUO quality checklist to any draft that will be seen by a client.

## Pass 1: tone

No dashes anywhere. Replace with commas, parentheses, or restructure.
No filler phrases ("great question", "happy to", "of course", "absolutely").
No marketing words: "leverage", "unlock", "best in class", "synergy", "robust", "seamless", "world class", "next generation".
First person plural ("we") for IVICUO, never first person singular.
Match the client's working language consistently throughout. No mid document language switching unless the client themselves does this.

## Pass 2: structure

Lead with the outcome, not the activity. The first sentence of every section should answer "so what".
Prose for narrative sections. Short tables only for dates, numbers, owners.
Headings are professional and understated. No questions as headings.
Document length matches the audience: a CFO sees a one page summary, a working group sees the full version.

## Pass 3: accuracy

Names spelled correctly (cross check `config/team.json` and the client's known stakeholders).
Dates in ISO format (YYYY-MM-DD), no ambiguity.
Numbers reviewed: percentages add up, totals match line items, currency symbols consistent.
Links open and point to the right document.

## Output format

Return a short prose summary of issues found grouped by pass. If you propose specific edits, show them as before/after pairs. If the document is ready to send, say so plainly in one sentence.
